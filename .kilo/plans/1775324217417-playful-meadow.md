# Plan: Rebuild GenshinRichPresence.exe Correctly

## Goal
Create a folder-based exe (not single-file) that is easy to auto-update from GitHub releases. The exe should be able to find its resources next to itself.

## User Preference
- **Folder-based exe** preferred over single-file
- Easier auto-update: can replace individual files (CSV, configs, etc.) from GitHub releases
- Resources can be picked up from the folder next to the exe

## Current Issue
The previous build was created with PyInstaller but data files (data/, CONFIG.py, configs) were manually copied AFTER the build - this is incorrect.

## Plan

### Step 1: Clean up the build folder
- Delete `builds/release/` contents
- Start fresh

### Step 2: Build folder-based exe (without --onefile)
Use PyInstaller WITHOUT `--onefile` flag:
```
pyinstaller --name GenshinRichPresence --windowed --add-data "gui;gui" --add-data "core;core" --add-data "data;data" gui_launcher.py
```
This creates:
```
dist/
└── GenshinRichPresence/
    ├── GenshinRichPresence.exe
    ├── gui/ (bundled)
    ├── core/ (bundled)
    ├── data/ (bundled)
    └── other dependencies...
```

### Step 3: Handle config files separately
Since config files (shared_config.json, gui_config.json, CONFIG.py) need to be editable by users:
- Keep them as external files in the release folder
- Users can edit them directly
- New releases can replace these files

### Step 4: Verify the build
- Check that exe exists in `dist/GenshinRichPresence/` folder
- Verify it runs properly

### Step 5: Final release structure
The final `builds/release/` should contain:
```
builds/release/
├── dist/
│   └── GenshinRichPresence/
│       ├── GenshinRichPresence.exe
│       ├── gui/
│       ├── core/
│       └── data/
├── shared_config.json (external, editable)
├── gui_config.json (external, editable)
└── CONFIG.py (external, editable)
```

This allows easy updating - just replace individual files from GitHub releases.
